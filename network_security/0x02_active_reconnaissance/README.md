# Active Reconnaissance

Active reconnaissance is the process of directly interacting with a target system or network to collect information such as open ports, running services, domain records, and possible entry points. Unlike passive reconnaissance, active reconnaissance sends requests to the target, so it can be detected by firewalls, logs, or security monitoring tools.

## Purpose

The goal of active reconnaissance is to understand how a target is exposed on the network and identify useful technical details for security testing. This step is commonly used in ethical hacking, penetration testing, and system administration to discover weaknesses before attackers can abuse them.

## Common Techniques

Active reconnaissance may include port scanning, service detection, DNS queries, ping sweeps, banner grabbing, and web directory discovery. Tools such as Nmap, RustScan, WHOIS, dig, and curl can help gather this information in a structured way.

## Important Note

Active reconnaissance should only be performed on systems you own or have permission to test. Scanning or probing systems without authorization can be illegal and may cause security alerts.
